官网 master-slave

mongodb的主从要注意的事项

  1. 主服务器挂掉，从服务器不会自动取代主服务器。
  2. 主服务器挂掉后重启，从服务器则会重新同步数据，之前同步过来的数据被初始化。
  3. 综合以上两点，mongodb要随时做好数据备份，以免造成无法弥补的数据丢失！！！

master

docker run -v ~/docker/test/mongo/test1/data/:/data/db -p 27001:27017 --name master_mongo --network docker_default mongo:3.2 mongod --dbpath /data/db --master

slave

 docker run -v ~/docker/test/mongo/test2/data/:/data/db -p 27002:27017 --name slave_mongo --network docker_default  mongo:3.2 mongod --dbpath /data/db --slave --source 192.168.1.222:27001
  • –only 可指定database
  • –slavedelay
  • –fastsyn
  • –aoturesync
  • –oplogSize

也可先创建从节点(slave)

docker run -v ~/docker/test/mongo/test3/data/:/data/db -p 27003:27017 --name slave1_mongo --network docker_default  mongo:3.2 mongod --dbpath /data/db --slave

然后再指定master

 use local
 db.sources.insert({host:"192.168.1.222:27001"})

主节点插入数据

db.test.insert({key:'insert 2'})

从节点可查询到

db.test.find().pretty()

slave链接查询报错问题

  • 2.6解决办法：

    db.setSlaveOk()
show dbs

  • 3.0解决办法：

    rs.slaveOk()
show dbs
